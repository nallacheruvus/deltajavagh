🔧 Database Setup & Configuration

Prerequisites

  • MySQL Server installed and running
  • MySQL Connector/J JDBC driver
  • Java 14 or higher

Database Configuration

The application is configured to connect to a MySQL database with the following details:

Configuration Item Value Description
Database Name deltajavamarch Primary database for the application
Host localhost Local machine (127.0.0.1)
Port 3306 Default MySQL port
Driver Class com.mysql.cj.jdbc.Driver MySQL Connector/J JDBC Driver
Username root Database user account
Password admin User password

Connection URL

jdbc:mysql://localhost:3306/deltajavamarch

🔌 Connection Parameters

Interface Constants (IFaceDbUtil.java)

public String className = "com.mysql.cj.jdbc.Driver";

The JDBC driver class for MySQL connections.

public String url = "jdbc:mysql://localhost:3306/deltajavamarch";

The complete JDBC connection URL including host, port, and database name.

public String uname = "root";

The database username for authentication.

public String password = "admin";

The database password for authentication.

⚠️ Security Note: In production environments, never hardcode database credentials. Use environment variables or configuration files with restricted access.

🔍 Available Queries

Query Overview

The application provides six predefined SQL queries for different data retrieval scenarios:

1. Basic Query - All Records

Query Name: query

Purpose: Retrieve all records from the register table

SELECT * FROM register;

Usage: Returns all user registration records from the database.

2. Descending Order Query

Query Name: querya

Purpose: Retrieve all records sorted by ID in descending order

SELECT * FROM register ORDER BY id DESC;

Usage: Used by secQuery() method to display records from highest to lowest ID.

3. Odd ID Query

Query Name: queryb

Purpose: Retrieve records with odd ID values

SELECT * FROM register WHERE id IN (1,3,5,7,9);

Usage: Used by thirdQuery() method to filter odd-numbered IDs.

4. Even ID Query

Query Name: queryc

Purpose: Retrieve records with even ID values

SELECT * FROM register WHERE id IN (2,4,6,8,10);

Usage: Used by fourthQuery() method to filter even-numbered IDs.

5. Name Filter Query

Query Name: queryd

Purpose: Retrieve records where name starts with 'J'

SELECT * FROM register WHERE name LIKE 'J%';

Usage: Used by fifthQuery() method to filter names starting with letter 'J'.

6. View Query

Query Name: querye

Purpose: Retrieve records from the myyiew database view

SELECT * FROM myyiew;

Usage: Used by sixthQuery() method to retrieve formatted view data.

⚙️ Database Methods (DbUtilsCls)

Method 1: retRegs()

public List<Register> retRegs()

Description

Retrieves all Register records from the database using a PreparedStatement.

Implementation Details

  • Executes: SELECT * FROM register
  • Uses: PreparedStatement for safe query execution
  • Returns: List of Register objects
  • Error Handling: SQLException caught and printed

Example Usage

DbUtilsCls db = new DbUtilsCls(); List<Register> registers = db.retRegs(); registers.forEach(System.out::println);

Method 2: retList()

public List<MyView> retList()

Description

Retrieves MyView records from the database view (myyiew).

Implementation Details

  • Executes: SELECT * FROM myyiew
  • Uses: Statement for executing the query
  • Returns: List of MyView objects
  • Error Handling: SQLException caught and printed

Example Usage

DbUtilsCls db = new DbUtilsCls(); List<MyView> views = db.retList(); views.forEach(System.out::println);

Method 3: retRegsProc()

public List<Register> retRegsProc()

Description

Retrieves Register records by executing a stored procedure.

Implementation Details

  • Executes: Stored procedure myproc
  • Uses: CallableStatement for executing procedures
  • Returns: List of Register objects
  • Error Handling: SQLException caught and printed

Example Usage

DbUtilsCls db = new DbUtilsCls(); List<Register> registers = db.retRegsProc(); registers.forEach(r -> System.out.println(r.id() + ": " + r.name()));
ℹ️ Note: The stored procedure myproc must exist in the database.

💡 Usage Examples

Example 1: Getting All Registers

DbUtilsCls db = new DbUtilsCls(); List<Register> allRegisters = db.retRegs(); for (Register reg : allRegisters) { System.out.println("ID: " + reg.id()); System.out.println("Name: " + reg.name()); System.out.println("Email: " + reg.email()); System.out.println("Mobile: " + reg.mobile()); System.out.println("---"); }

Example 2: Filtering with Streams

DbUtilsCls db = new DbUtilsCls(); List<Register> registers = db.retRegs(); // Filter names starting with 'J' registers.stream() .filter(r -> r.name().startsWith("J")) .forEach(System.out::println);

Example 3: Statistical Analysis

DbUtilsCls db = new DbUtilsCls(); List<Register> registers = db.retRegs(); var summary = registers.stream() .map(Register::id) .collect(Collectors.summarizingInt(i -> i)); System.out.println("Max ID: " + summary.getMax()); System.out.println("Min ID: " + summary.getMin()); System.out.println("Average ID: " + summary.getAverage()); System.out.println("Count: " + summary.getCount()); System.out.println("Sum: " + summary.getSum());

Example 4: Sorting Records

DbUtilsCls db = new DbUtilsCls(); List<Register> registers = db.retRegs(); // Sort by ID descending registers.stream() .sorted((a, b) -> b.id() - a.id()) .forEach(System.out::println);

Example 5: View Data Retrieval

DbUtilsCls db = new DbUtilsCls(); List<MyView> views = db.retList(); views.forEach(view -> { System.out.println("Name: " + view.name()); System.out.println("Email: " + view.email()); System.out.println("Mobile: " + view.mobile()); System.out.println("Address: " + view.raddress()); System.out.println(); });

📊 Database Schema

Register Table

CREATE TABLE register ( id INT PRIMARY KEY, name VARCHAR(100), email VARCHAR(100), mobile VARCHAR(20) );

MyView Database View

CREATE VIEW myyiew AS SELECT id, name, email, mobile, address AS raddress FROM register;

Stored Procedure

CREATE PROCEDURE myproc() BEGIN SELECT * FROM register; END;

Data Structure

Column Type Description
id INT User unique identifier (Primary Key)
name VARCHAR(100) User's full name
email VARCHAR(100) User's email address
mobile VARCHAR(20) User's phone number
✓ Sample Data: The register table contains sample user data that can be queried using the predefined queries or through custom SQL statements.
